# 문서 스타일 가이드

한국어 단일 문서 체계를 안정적으로 유지하기 위한 운영 기준입니다.

## 기본 원칙

- 문서 설명문은 한국어 중심으로 작성합니다.
- 핵심 기술어는 원어를 유지합니다.
- 신규 문서/링크는 기본 `.md` 경로만 사용합니다.
- 저작 가이드는 `.gyeol` 기준으로 작성하고, JSON IR은 산출물/도구 연동 맥락에서만 설명합니다.
- 규칙 위반은 CI에서 실패로 처리합니다.

## 표기 규칙

### 혼합 표기 원칙

- 기술어 첫 등장 시 `한국어 설명 + 원어` 또는 `원어 + 한국어 설명` 중 하나를 사용합니다.
- 동일 문서에서 같은 개념은 같은 표기로 반복합니다.
- 코드 심볼/API 이름(`Runner::step`, `StepType`, `Variant`)은 원문을 유지합니다.

### 헤더 규칙

- 일반 가이드/설명 문서의 제목은 한국어 중심으로 작성합니다.
- API 문서의 타입/클래스명은 영문 유지가 가능합니다.
- 섹션 제목의 영문 단독 표기는 피하고, 필요한 경우 괄호 병기를 사용합니다.

## 용어집

| 권장 표기 | 비권장 표기 | 예시 |
| --- | --- | --- |
| `JSON IR` | `JSONIR` | `canonical JSON IR` |
| `locale catalog` | `locale-catalog` (설명문) | `locale catalog v2` |
| `저장/로드` | `세이브/로드` | `저장/로드 호환성` |
| `Runner` | `런너` | `Runner VM` |
| `fallback` | `fall back` | `fallback 선택지` |

설명:
- `locale-catalog`는 CLI 옵션/포맷 문자열처럼 변경 불가능한 코드/값에서는 예외입니다.
- 용어집의 비권장 표기는 문서 설명문 기준으로 자동 검사 대상입니다.

## 경로/링크 규칙

- 언어 접미사 기반 파일명 경로를 사용하지 않습니다.
- 내부 문서 링크는 상대 경로 `.md` 형식으로 작성합니다.
- 가능한 한 대상 문서의 실제 섹션 앵커까지 함께 검증합니다.

## 자동 검사 항목

문서 품질 게이트(`tools/dev/check-docs-policy.py`)는 다음을 검사합니다.

- 언어 접미사 파일 경로 참조 금지
- `mkdocs.yml` nav 경로 실존 및 `.md` 규칙
- 문서 내 로컬 링크/앵커 기본 정합성
- 코드블록 외 과도한 영문 연속 문장
- 비권장 용어 패턴

운영 절차는 [릴리즈/PR 정책의 문서 운영 체크리스트](advanced/release-policy.md#문서-운영-체크리스트)를 기준으로 점검합니다.
